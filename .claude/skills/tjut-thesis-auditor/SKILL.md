# 天津工业大学本科毕业论文 Word 排版 Skill

## 目标

将 `thesis.md` 转换为符合天津工业大学本科毕业论文格式要求的 `.docx` 文件，并输出格式审查报告。

不要只做普通 Markdown 转 Word。必须以 `template/本科毕业论文格式模板.docx` 为格式母版，结合 `reference/前辈参考论文.pdf` 的实际排版习惯，使用 Python 脚本和 python-docx / lxml 生成最终 Word。

## 输入文件

- `thesis.md`：论文 Markdown 正文
- `template/*.docx`：学校 Word 格式模板
- `reference/*.pdf`：前辈参考论文，用于观察实际章节、图表、目录、页眉页脚效果
- `assets/figures/`：论文插图、流程图、系统图、截图
- 项目文档：README.md、DOCS.md、COURSE_METHODOLOGY.md、SKILLS.md，用于核对项目事实

## 输出文件

- `output/张朔_本科毕业论文_排版稿.docx`
- `output/格式审查报告.md`
- 可选：`output/缺失图片和表格清单.md`

## 必须遵守的格式规则

1. 封面包含题目、学号、姓名、专业、学院、指导教师、职称、完成日期。
2. 摘要标题居中，中文摘要正文小四号宋体；关键词顶格，关键词之间用分号。
3. Abstract 使用 Times New Roman，标题加粗居中；Keywords 顶格。
4. 目录标题居中，目录二字中间空两格；目录由 Word 标题样式自动生成。
5. 正文章标题如“第一章 绪论”：小三号黑体，居中，段前段后 0.5 行。
6. 二级标题如“一、研究背景”：四号黑体。
7. 三级标题如“（一）研究背景”：小四号黑体。
8. 正文：中文宋体小四，英文 Times New Roman，小四，1.5 倍行距，首行缩进 0.84 厘米。
9. 每章另起一页。
10. 图题位于图下方，宋体五号居中，编号连续，如“图 3.1 系统用例图”。
11. 表题位于表上方，宋体五号居中，编号连续，如“表 4-1 平台功能模块表”。
12. 表格尽量处理为三线表：顶线、栏目线、底线，无左右竖线。
13. 参考文献另起一页，编号连续，五号宋体 / Times New Roman，悬挂缩进。
14. 附录、致谢另起一页。
15. 页眉为“天津工业大学 2026 届本科生毕业设计（论文）”，正文页码从正文开始。
16. 插图应居中，图片比例不得变形。

## 工作流程

### P0：先审查，不生成 Word

先读取 `thesis.md`，输出：
- 标题层级问题
- 图表编号问题
- 图片路径缺失问题
- 表格过宽问题
- 参考文献格式问题
- 需要人工补充的内容

### P1：生成排版脚本

创建或更新 `scripts/build_thesis_docx.py`，不要手工复制粘贴生成 docx。
脚本应完成：
- 解析 Markdown
- 建立 Word 样式
- 套用模板页眉页脚
- 插入封面、摘要、Abstract、目录、正文、参考文献、致谢
- 处理图片、图题、表题、三线表
- 插入分页符
- 输出 docx

### P2：运行脚本生成 docx

运行脚本，生成 `output/张朔_本科毕业论文_排版稿.docx`。

### P3：格式审查

对生成文件再次检查，输出 `output/格式审查报告.md`，明确哪些项目已完成、哪些需要人工打开 Word 更新目录或微调。

## 注意

- 不要把论文内容改写成另一个主题。
- 不要删除作者原有章节。
- 不要伪造参考文献。
- 不要把 API Key 写入任何输出文件。
- 目录页码可以插入 Word 域，最终由用户在 Word 中右键“更新域/更新整个目录”。
